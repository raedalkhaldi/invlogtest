🚀 Cursor AI Guidance for Social Check-in App Development

🔹 Project Overview

This document serves as a structured guide for Cursor AI to develop a social check-in app where users can follow each other, check-in to restaurants, attach optional photos, and interact through likes & comments. The app must be flexible for future changes.

🔹 Backend Setup & Structure

✅ 1️⃣ Use Firebase for Simplicity

  • Authentication: Firebase Authentication (Google & Email login).
  • Database: Firestore for storing check-ins, users, and interactions.
  • Storage: Firebase Storage for handling profile pictures & check-in images.
  • Notifications: Firebase Cloud Messaging (FCM) for real-time updates.

✅ 2️⃣ App Structure for Flexibility

Ensure the app is structured in a way that allows easy feature expansion.

  • Follow MVVM (Model-View-ViewModel) pattern.
  • Use Provider or Riverpod for state management.
  • Separate logic from UI for better maintainability.

🔹 Key Features to Implement

✅ 4️⃣ User Authentication

  • Allow Google & Email login.
  • Ensure secure session management.

✅ 5️⃣ Check-Ins & Timeline

  • Users can check in with restaurant location (GPS-based).
  • Optional photo upload with Firestore integration.
  • Display timeline of friends’ check-ins.

✅ 6️⃣ Likes & Comments

  • Users can like & comment on check-ins.
  • Firestore should handle real-time updates.

✅ 7️⃣ Profile Management

  • Each user has a profile page.
  • Users can update bio & profile picture.

✅ 8️⃣ Push Notifications

  • Notify users when they receive likes or comments.
